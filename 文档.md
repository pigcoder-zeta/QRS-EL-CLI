## 1. 项目简介

QRS-EL 是面向**表达式注入漏洞**（EL Injection / SSTI）的全自动安全检测平台：

| Agent | 职责 |
|-------|------|
| **Agent-Q** | 生成 CodeQL 污点追踪规则，内置自修复循环（最多 MAX_RETRIES 次） |
| **Agent-R** | 语义审查 CodeQL 发现，LLM 过滤误报并输出置信度评分 |
| **Agent-S** | 对确认漏洞生成可直接使用的 HTTP PoC 与 Payload 列表 |

支持漏洞类型：**Spring EL（SpEL）/ OGNL / MVEL**（Java）；**Jinja2 / Mako**（Python）。

## 2. 系统架构

```
输入（本地目录 / GitHub URL）
        │
        ▼
┌─────────────────────────────────────────────────┐
│               Coordinator（调度器）               │
│                                                  │
│  Phase 0  GithubRepoManager                      │ 克隆 + 构建探测
│  Phase 1  CodeQLRunner + DatabaseCache           │ 建库（Git-Hash 缓存）
│  Phase 2  Agent-Q + QLTemplateLibrary            │ 规则生成（模板优先）
│         → RuleMemory (Few-Shot + 神经嵌入检索)    │ 历史规则增强
│  Phase 3  CodeQLRunner.analyze()                 │ SARIF 扫描
│  Phase 4  Agent-R（语言感知提示词）               │ 误报过滤
│  Phase 5  Agent-S（内置 Payload 策略库）          │ PoC 生成
└─────────────────────────────────────────────────┘
        │
        ▼
输出：SARIF + JSON 报告 + HTML 报告 + 扫描历史
```

### 并行扫描（建库一次，共享数据库）

```
Coordinator.run_parallel(vuln_types=[...])
       │
       ├─ 主线程：Phase 0+1（建库唯一一次）
       │
       └─ ThreadPoolExecutor:
           ├─ Thread 1: Phase 2-5 → Spring EL Injection
           ├─ Thread 2: Phase 2-5 → OGNL Injection
           └─ Thread 3: Phase 2-5 → MVEL Injection
```

## 4. 安装与环境配置

### 前置依赖

- Python 3.10+
- [CodeQL CLI](https://github.com/github/codeql-cli-binaries/releases)（需在 PATH 中可访问）
- Git（GitHub 模式需要）
- Java JDK（分析 Java 项目时需要）

### 安装

```powershell
pip install -r requirements.txt
```

主要依赖：

| 包 | 用途 |
|----|------|
| `langchain-openai` | LLM 调用 |
| `GitPython` | 仓库克隆 |
| `scikit-learn` | TF-IDF 规则相似度 |
| `pyyaml` | YAML 配置文件 |
| `rich` | 终端进度条与彩色输出 |
| `sentence-transformers`（可选） | 神经嵌入增强规则检索 |

### 配置 API Key

```powershell
copy .env.example .env
# 编辑 .env，填入 OPENAI_API_KEY
```

## 5. 快速开始

```powershell
# 环境检查
python -m src.main --check

# 列出内置 QL 模板
python -m src.main --list-templates

# 本地目录扫描
python -m src.main --source-dir ./target --language java `
    --vuln-type "Spring EL Injection" `
    --output-html data/results/report.html

# GitHub URL 扫描
python -m src.main --github-url https://github.com/WebGoat/WebGoat `
    --language java --vuln-type "Spring EL Injection"

# 多漏洞并行扫描（建库一次）
python -m src.main --github-url https://github.com/WebGoat/WebGoat `
    --language java `
    --vuln-types "Spring EL Injection" "OGNL Injection" "MVEL Injection" `
    --output-json data/results/report.json `
    --output-html data/results/report.html

# 使用 YAML 配置文件
python -m src.main --config qrs-el.yml
```

### 常用参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--max-retries N` | 3 | Agent-Q 自修复最大次数 |
| `--min-confidence F` | 0.6 | Agent-R 置信度过滤阈值 |
| `--parallel-workers N` | 3 | 并行扫描线程数 |
| `--output-json PATH` | — | 导出 JSON 报告 |
| `--output-html PATH` | — | 导出 HTML 报告 |
| `--no-agent-r` | — | 跳过语义审查 |
| `--no-agent-s` | — | 跳过 PoC 生成 |
| `--no-rule-memory` | — | 禁用规则记忆库 |
| `--no-cleanup` | — | 保留克隆目录 |
| `--config PATH` | — | YAML 配置文件 |
| `--list-templates` | — | 列出内置模板 |
| `--check` | — | 环境健康检查 |

## 6. 各模块说明

### Agent-Q：规则合成器

- **双语言黄金模板**：Java（MethodCall/hasQualifiedName）与 Python（Call/Attribute）各一套系统提示，编译成功率大幅提升
- **模板优先策略**：`QLTemplateLibrary` 内置 7 个预验证模板（Java 4 个 + Python 3 个）
- **Few-Shot 注入**：从 `RuleMemory` 检索相似历史规则注入 Prompt
- **自修复循环**：编译失败 → LLM 修复 → 最多 `MAX_RETRIES` 次

### Agent-R：语义审查器

- **语言感知 prompt**：Java 检查 `StandardEvaluationContext` vs `SimpleEvaluationContext`；Python 检查 `SandboxedEnvironment`、`escape()` 净化
- 代码块语法高亮随语言切换（`java`/`python`）
- 输出 `VulnStatus`（VULNERABLE/SAFE/UNCERTAIN）+ 置信度 + 推理说明

### Agent-S：PoC 生成器

- 内置 Payload 策略库（SpEL/OGNL/MVEL/Jinja2/Mako 共 20+ 条载荷）
- LLM 推断 HTTP 触发接口 + 参数名 + 完整 curl 示例
- 输出结构化 `PoCResult`

### RuleMemory：规则记忆库（Phase 6 升级）

- **双重检索引擎**：优先使用 `sentence-transformers` 神经嵌入；未安装时自动降级为 scikit-learn TF-IDF
- 嵌入矩阵持久化为 `embeddings.npz`，避免重复编码
- 新增规则时自动使嵌入缓存失效

### HTML 报告（Phase 7）

- 完全自包含（无外部 CDN 依赖），深色主题，支持离线查看
- 包含统计卡片、分漏洞类型结果表格、Agent-R 置信度条形图、Agent-S PoC 详情
